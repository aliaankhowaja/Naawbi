Auth Flow — 3 Architectural Approaches

How should login/registration fit into the JavaFX app lifecycle?

A

Scene Swap (Recommended)

App starts → shows Login scene on the main Stage → on success, swaps scene to CourseCatalog on the same Stage. One window, no flicker, clean lifecycle.

Main.java
└─ start() → load LoginView.fxml
LoginController → on success
└─ stage.setScene(catalogScene)
└─ stage.setTitle("Naawbi")

Pros

  • Simplest lifecycle
  • One stage, no race conditions
  • Matches existing pattern (Main.java already does this)

Cons

  • Slight resize if login is smaller than 1920×1080
B

Separate Login Stage

App starts → opens a small login Stage → on success, closes it and opens the main Stage. Common in desktop apps.

Main.java
└─ start() → new Stage() → LoginView
LoginController → on success
└─ loginStage.close()
└─ new Stage() → CourseCatalogView

Pros

  • Login can be a small centered window
  • Clean visual separation

Cons

  • Two stages to manage
  • Window flash on transition
  • More boilerplate
C

Modal Dialog over App

App loads catalog in background → immediately shows a blocking modal dialog for login. Dialog closes on success, revealing the catalog.

Main.java
└─ start() → load CourseCatalogView
└─ show LoginDialog (blocking)
LoginDialog → on success
└─ dialog.close() → catalog visible

Pros

  • No scene transition needed

Cons

  • Catalog loads behind login (wrong)
  • Security smell — data accessible before auth
  • Not recommended for RBAC
